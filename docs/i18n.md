# Internationalization

BMZ Player supports the following UI locales:

- `ja` — 日本語
- `en` — English
- `ko` — 한국어
- `zh-CN` — 简体中文
- `zh-TW` — 繁體中文（台灣）
- `zh-HK` — 繁體中文（香港）

The profile setting `ui.language` stores one of these canonical BCP 47 codes.
Locale input is normalized before use: language variants of Japanese, English,
and Korean map to their base code; Simplified Chinese maps to `zh-CN`;
Traditional Chinese maps to `zh-TW`; and Hong Kong or Macao Chinese maps to
`zh-HK`. Unknown values fall back to `ja`.

## Scope

BMZ-owned desktop UI, application-generated notifications, and the IR website
are localized. The desktop and website catalogs are separate because their UI
and message formatting needs differ.

The following content is never translated:

- chart metadata such as title, artist, genre, and subtitle;
- course, difficulty table, user, and provider names;
- file paths, URLs, database values, replay data, IR payloads, and signed JSON;
- third-party or user skin text, images, property names, and file options;
- developer logs, internal error context, and license text.

The bundled default skin and the CLI are intentionally outside the current
i18n milestone. Gameplay terms such as `PGREAT`, `FAST`, `SLOW`, `EX SCORE`,
`BP`, and `BPM` remain stable English terms unless this document explicitly
defines a translated form later.

## Fallback

Catalogs must contain the same message keys and parameters. Runtime fallback
is only a safety net:

- `zh-HK` -> `zh-TW` -> `en` -> `ja`
- `zh-TW`, `zh-CN`, `ko` -> `en` -> `ja`
- `en` -> `ja`
- `ja`

Persistence and protocol values must not depend on the active locale. Display
labels are resolved at render time and must not be used as widget IDs, list
identity, database keys, or serialized enum values.

## Catalog maintenance

Desktop messages live in `crates/bmz-player/src/i18n/locales/`. The Japanese
catalog is the reference signature; tests require every desktop catalog to use
the same keys and Fluent parameters. The IR website keeps its independent
catalogs in the repository-level `i18n/locales/` directory and validates them
through the Nuxt type check and IR test suite.

When adding a message, use a semantic key, pass dynamic values as formatter
arguments, and keep chart/skin/user-authored values as arguments rather than
copying them into a catalog. A missing desktop key is shown as the key itself so
that incomplete localization is visible during development.
